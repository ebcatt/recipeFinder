# Recipe Finder

## What this is

This is a static recipe finder website I built using HTML, CSS, and JavaScript. The idea was to let users search for recipes and get results dynamically through an API instead of just displaying hardcoded content.

## Why I built it

I built this project to get more practice with JavaScript and to make something that felt a little more interactive than a basic static site. I wanted to work with API data in a way that was actually useful and easy to understand.

## What it does

* Lets users search for recipes
* Pulls recipe data from an API
* Displays results dynamically on the page
* Gives users a cleaner way to browse recipe ideas than just scrolling through static content

## Tech used

* HTML
* CSS
* JavaScript
* Recipe API integration

## How to run it

1. Clone this repository
2. Open the project folder
3. Open `index.html` in your browser

Depending on the API setup, you may also need your own API key or updated configuration to make requests work correctly.

## What I learned

This project helped me get more comfortable with JavaScript, especially working with APIs, handling returned data, and updating the page dynamically. It also gave me more practice connecting front-end code to outside data instead of building everything manually.

## Challenges

The hardest part was getting the API calls and returned data to work the way I wanted. It took some trial and error to make sure the results showed up correctly and that the site still felt organized instead of messy.

## If I kept working on it

I’d like to:

* Add more than one API source
* Improve filtering and search options
* Make the layout feel even more polished
* Add better error handling for missing results or failed requests

## Notes

This project is a good example of where I started moving beyond basic front-end pages and into more interactive web development. It’s simple, but it helped me build confidence working with real data and JavaScript functionality.
